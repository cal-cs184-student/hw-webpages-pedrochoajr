Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm is used to produce Bezier curves. This is done by starting off with a set of n points. From these n points, we apply linear interpolation on the points with a parameter t(where t is between 0 and 1) and end up with n-1 points. After repeating the linear interpolation on the set of points created by the previous round of linear interpolation, we end up with a single point. This single point is actually on the Bezier curve, so to produce the entire curve, rather than a single point, we must repeat this process with several values of t to obtain the curve.

The implementation of de Casteljau's algorithm was very simple and straightforward to implement since it was broken down to just doing a single step. The base case is that there are at least 2 points because linear interpolation requires a pair of points for it to work. After defining my base case I noted that the linear interpolation function is (1-t)*points[i] + t*points[i+1] where i is the index of a array called points, which holds all the points given for that step. I interpolate two pairs of nodes and store the result in a return array. I repeated this until I went through all the elements in points. This makes it so that if I begin with n points, then by the end of the step, I have n-1 new points from interpolation, and am able to repeat this call until I have a single point.

Part 2: Bezier surfaces with separable 1D de Casteljau

De Casteljau's algorithm extends naturally from Bezier curves to Bezier surfaces. Since Bezier curves are surfaces, we also have to apply the interpolation in both dimensions. In Bezier curves, we use the parameter t, but to accommodate for the second dimension, we will use the parameters u and v. First, I evaluated the Bezier curve(using de Casteljau's algorithm) along one dimension using the parameter u. I then applied a Bezier curve to the result of the previous step with the parameter v. This produces a single point on the Bezier surface.

The implementation consisted of 3 main components:
1. evaluateStep(points, t), identically to part 1 it returns a step of De Casteljau's algorithm
2. evaluate1D(points, t), this runs the De Casteljau's algorithm, by calling evaluateStep repeatedly
3. evaluate(u, v), returns point on bezier surface

In evaluate (u, v), it first evaluates each row of the controlPoints at parameter u using the evaluate1D function. This produces a set of points that is stored in a vector called collapsedRows. I then call evaluate1D again on collapsedRows at parameter v to obtain the final point, which will be the return value of evaluate.

Part 3: Area-weighted vertex normals

In my implementation to compute the area-weighted normal vectors at vertices, I traversed the faces incident to each vertex using the halfedge structure. I got my starting position by first calling halfedge(). From there, I set up my loop so that I get each point using h->vertex()->position, h->next()->vertex()->position, h->next()->next()->vertex()->position. This would return the three vertices of the current face. With these three points, I constructed two edge vectors from the vertex we are traveling around. While preparing for this question and my previous classes, I found that the cross product of two triangle edges produces a vector whose magnitude is proportional to the triangle's area while still in the direction of the face normal. With this information in mind, I knew I would get the same information as finding the triangle area. I summed the cross product into a vector called returnVector. I then repeated this process for each face around the main vertex. I made sure to stop once I returned to the starting face, and skip when h->face()->isBoundary(). After visiting all adjacent faces, I normalized the returnVector to obtain a unit normal, which was then returned.
